### Bug A: installation.js 安装完成未更新订单状态+未触发佣金
**位置**: `src/backend/src/routes/installation.js` PUT /:id

**问题**: 当安装任务 status='completed' 时，只更新了 installation_task 本身，没有：
1. 更新 order_master.order_status = 'installed'
2. 调用 commissionService.autoGenerate()

**修复**:
```javascript
} else if (status === 'completed') {
  updates.push(`actual_complete = NOW()`);
  const taskRow = await db.query(
    `SELECT order_id FROM installation_task WHERE id = $1`, [id]
  );
  if (taskRow.rows.length > 0) {
    const orderId = taskRow.rows[0].order_id;
    await db.query(
      `UPDATE order_master SET order_status = 'installed', updated_at = NOW() WHERE id = $1`,
      [orderId]
    );
    try {
      require('../services/commissionService').autoGenerate(orderId);
    } catch (e) {
      logger.error('安装完成触发佣金失败:', e.message);
    }
  }
}
```

**验证**: DB查询确认3个测试订单全部正确更新为 installed 状态 ✅

## 新发现并修复的Bug #4

### Bug D: delivery 路由缺少 COMMIT（事务未提交）
**位置**: `src/backend/src/routes/order.js` POST /:id/delivery

**问题**: 物流签收API开启事务（BEGIN），执行了所有操作，但**从未 COMMIT**。`client.release()` 仅归还连接池，导致数据在返回200后被回滚。安装任务ID虽然返回给了客户端，但数据库中实际不存在。

**根本原因**: `res.json()` 发送响应后，异常处理分支的 `finally { client.release() }` 释放连接时，悬空事务自动ROLLBACK。

**修复**: 在 `res.json()` 前加 `await client.query('COMMIT')`

```javascript
// res.json() 之前
await client.query('COMMIT');
res.json({ success: true, ... });
```

**验证**: DB查询确认安装任务已正确创建，订单状态成功变为 installed ✅
