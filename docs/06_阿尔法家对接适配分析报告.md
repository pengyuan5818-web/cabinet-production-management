### 2.2 发现的设计差距

#### 差距1：拆单结果数据

**阿尔法家输出**:
```json
{
  "cabinet_no": "G001",
  "board_list": [
    {"board_name": "侧板", "length": 1200, "width": 600, "thickness": 18, "qty": 2},
    {"board_name": "底板", "length": 1160, "width": 580, "thickness": 18, "qty": 1}
  ],
  "hardware_list": [
    {"name": "铰链", "spec": "快装", "qty": 6},
    {"name": "导轨", "spec": "隐藏式", "qty": 3}
  ],
  "hole_data": [...],
  "edge_data": [...]
}
```

**本系统现有设计**:
- `order_detail`: 只有产品级别
- 缺少板件级别明细

**建议修改**:
```sql
-- 新增：拆单板件表
CREATE TABLE cabinet_board (
  id UUID PRIMARY KEY,
  order_detail_id UUID FK,      -- 关联订单明细
  cabinet_no VARCHAR(50),        -- 柜体编号
  board_name VARCHAR(100),       -- 板件名称
  board_type VARCHAR(50),        -- 板件类型：侧板/底板/门板等
  length INT,                    -- 长度(mm)
  width INT,                     -- 宽度(mm)
  thickness DECIMAL(5,2),        -- 厚度(mm)
  quantity INT,                  -- 数量
  material VARCHAR(100),         -- 材料
  color VARCHAR(50),             -- 颜色
  edge_band VARCHAR(100),        -- 封边要求
  hole_data JSON,                -- 孔位数据
  cut_data JSON,                 -- 切割数据
  alpha_imported BOOLEAN,        -- 是否来自阿尔法家
  alpha_file_no VARCHAR(50),      -- 阿尔法家文件编号
  created_at DATETIME
);

-- 修改：订单明细表增加板件关联
ALTER TABLE order_detail ADD COLUMN has_boards BOOLEAN;
ALTER TABLE order_detail ADD COLUMN board_count INT;
```

#### 差距2：生产工序细化

**阿尔法家特有工序**:
- 折弯加工（不锈钢特有）
- 焊接打磨（不锈钢特有）
- 台面粘接
- 水槽开孔

**本系统现有工序**:
- 14阶段标准工序

**建议修改**:
```sql
-- 扩展订单追踪阶段枚举
ALTER TYPE order_stage ADD VALUE 'bending';        -- 折弯
ALTER TYPE order_stage ADD VALUE 'welding';         -- 焊接
ALTER TYPE order_stage ADD VALUE 'polishing';       -- 打磨
ALTER TYPE order_stage ADD VALUE 'countertop';      -- 台面安装
ALTER TYPE order_stage ADD VALUE 'sink_cut';         -- 水槽开孔
```

#### 差距3：材料清单(BOM)结构

**阿尔法家BOM**:
```json
{
  "cabinet_no": "G001",
  "boards": [...],
  "hardware": [
    {"code": "H001", "name": "铰链", "spec": "快装", "brand": "百隆", "qty": 6}
```

**建议修改**:
```sql
-- 新增：订单BOM表
CREATE TABLE order_bom (
  id UUID PRIMARY KEY,

  cabinet_no VARCHAR(50),
  bom_type ENUM('board', 'hardware', 'consumable'),
  item_code VARCHAR(50),

  spec VARCHAR(100),
  brand VARCHAR(100),
  unit VARCHAR(20),
  quantity DECIMAL(10,2),
  unit_price DECIMAL(12,2),
  from_alpha BOOLEAN,            -- 是否来自阿尔法家
```

## 三、冲突点分析与解决方案

### 3.1 数据重复问题

| 冲突 | 分析 | 解决方案 |
|------|------|---------|
| 客户信息 | 阿尔法家和本系统都存客户 | 以本系统为主，阿尔法家作为生产端 |
| 设计方案 | 酷家乐→阿尔法家→本系统 | 本系统作为数据归集中心 |
| 报价数据 | 阿尔法家可报价，本系统也有报价 | **使用本系统报价**，阿尔法家报价仅作参考 |

### 3.2 数据同步策略

```
┌─────────────┐    手动/定时    ┌─────────────┐
│   酷家乐    │ ───────────────►│   阿尔法家   │
│  (设计)     │    设计数据      │  (拆单)     │
└─────────────┘                └──────┬──────┘
                                      │ 生产数据
                                      │ 手动导出
                                      ▼
┌─────────────────────────────────────────────────┐
│                    本系统                        │
│                  (生产管理)                       │
│                                                 │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐         │
│  │ 生产追踪 │  │ 仓库管理 │  │ 财务管理 │         │
│  └─────────┘  └─────────┘  └─────────┘         │
└─────────────────────────────────────────────────┘
```

**同步方式**:
1. **设计阶段**: 酷家乐设计 → 导出到阿尔法家
2. **拆单阶段**: 阿尔法家拆单 → 导出Excel/JSON → 导入本系统
3. **生产阶段**: 本系统全程管理
4. **包装发货**: 本系统 → 导出到阿尔法家(可选)

### 3.3 对接流程设计

#### 3.3.1 阿尔法家 → 本系统（导入）

```javascript
// 阿尔法家导出数据导入流程
async function importFromAlpha(data) {
  // 1. 解析阿尔法家导出文件
  const alphaData = parseAlphaExport(data);
  
  // 2. 匹配订单
  const order = await matchOrder(alphaData.order_no);
  
  // 3. 导入板件数据
  for (const board of alphaData.boards) {
    await createCabinetBoard({
      order_id: order.id,
      cabinet_no: board.cabinet_no,
      board_name: board.board_name,
      length: board.length,
      width: board.width,
      thickness: board.thickness,
      quantity: board.quantity,
      from_alpha: true
    });
  }
  
  // 4. 导入五金数据 → 生成采购建议
  for (const hardware of alphaData.hardware) {
    await createOrderBOM({
      order_id: order.id,
      item_code: hardware.code,
      item_name: hardware.name,
      spec: hardware.spec,
      quantity: hardware.qty,
      from_alpha: true
    });
  }
  
  // 5. 更新订单状态为"已拆单"
  await updateOrderStatus(order.id, 'split_completed');
  
  // 6. 生成生产工单
  await generateProductionOrder(order.id);
}
```

| 1 | 新增 cabinet_board | 表 | 拆单板件明细 |
| 2 | 新增 order_bom | 表 | 订单BOM清单 |

| 5 | 修改 design_attachment | 字段 | 增加阿尔法家文件关联 |

### 4.2 新增功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| Alpha导入助手 | 解析阿尔法家Excel/JSON文件 | P0 |
| BOM对比 | 对比阿尔法家BOM与实际采购 | P1 |
| 板件追踪 | 按板件级别追踪生产进度 | P1 |
