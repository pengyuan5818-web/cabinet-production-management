### 1.2 数据权限隔离

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           数据权限模型                                        │
└─────────────────────────────────────────────────────────────────────────────┘

【工厂视角】
├── 可看所有经销商数据
├── 可看所有订单
├── 可看所有客户
├── 可看所有收付款
└── 可分配经销商权限

【经销商视角】
├── ❌ 不可看其他经销商数据
├── ✅ 只看自己上传的客户
├── ✅ 只看自己下单的订单
├── ✅ 只看自己应付/应收
└── ✅ 只看自己提交的报价

【客户视角】
└── ✅ 只看自己的订单进度（扫码/网页）
```

## 二、经销商账号体系

### 2.1 经销商类型

| 类型 | 说明 | 权限范围 |
|------|------|---------|
| **直营店** | 工厂直营 | 与工厂员工类似 |
| **特约经销商** | 签约经销商 | 客户管理+下单+查看生产 |
| **普通经销商** | 注册经销商 | 客户管理+下单 |
| **体验店** | 仅展示 | 仅查看产品+报价 |

### 2.2 经销商账号结构

```
经销商组织架构
│
├── 经销商总部
│   ├── 管理员账号 (1个)
│   ├── 财务账号 (1-2个)
│   ├── 销售账号 (不限)
│   └── 设计账号 (不限)
│
└── 经销商门店 (可选)
    ├── 门店管理员
    └── 门店销售
```

### 2.3 经销商主表

```sql
-- ============================================
-- 经销商主表
-- ============================================
CREATE TABLE dealer (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    dealer_code VARCHAR(50) NOT NULL UNIQUE COMMENT '经销商编号：JXS+序号',
    dealer_name VARCHAR(200) NOT NULL COMMENT '经销商名称',
    dealer_type VARCHAR(20) NOT NULL COMMENT '类型：direct/authorized/retail/experience',
    
    -- 基本信息
    legal_person VARCHAR(100) COMMENT '法人代表',
    contact_person VARCHAR(100) COMMENT '联系人',
    phone VARCHAR(20) NOT NULL COMMENT '联系电话',
    mobile VARCHAR(20) COMMENT '手机',
    email VARCHAR(100) COMMENT '邮箱',
    
    -- 地址
    province VARCHAR(50),
    city VARCHAR(50),
    district VARCHAR(50),
    address VARCHAR(500) COMMENT '详细地址',
    
    -- 营业执照
    business_license VARCHAR(500) COMMENT '营业执照图片',
    tax_no VARCHAR(50) COMMENT '税号',
    bank_name VARCHAR(100) COMMENT '开户银行',
    bank_account VARCHAR(50) COMMENT '银行账号',
    
    -- 合同信息
    contract_no VARCHAR(50) COMMENT '合同编号',
    contract_start DATE COMMENT '合同开始',
    contract_end DATE COMMENT '合同结束',
    commission_rate DECIMAL(5,2) COMMENT '佣金比例%',
    
    -- 状态
    status VARCHAR(20) DEFAULT 'pending' COMMENT '状态：pending/active/suspended/terminated',
    activation_date DATE COMMENT '激活日期',
    
    -- 额度
    credit_limit DECIMAL(12,2) DEFAULT 0 COMMENT '信用额度',
    credit_used DECIMAL(12,2) DEFAULT 0 COMMENT '已用额度',
    
    -- 上级
    parent_dealer_id UUID REFERENCES dealer(id) COMMENT '上级经销商',
    
    -- 系统账号
    admin_user_id UUID COMMENT '管理员用户ID',
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_dealer_status ON dealer(status);
CREATE INDEX idx_dealer_type ON dealer(dealer_type);
```

### 2.4 经销商用户表

```sql
-- ============================================
-- 经销商用户表
-- ============================================
CREATE TABLE dealer_user (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_no VARCHAR(50) NOT NULL COMMENT '用户工号',
    dealer_id UUID NOT NULL REFERENCES dealer(id),
    
    -- 账号信息
    username VARCHAR(50) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    email VARCHAR(100),
    phone VARCHAR(20),
    
    -- 人员信息
    name VARCHAR(100) NOT NULL COMMENT '姓名',
    id_card VARCHAR(20) COMMENT '身份证',
    position VARCHAR(50) COMMENT '职位',
    
    -- 角色
    role VARCHAR(20) NOT NULL COMMENT '角色：admin/finance/sales/designer',
    
    -- 权限范围
    permission_scope JSONB COMMENT '权限配置',
    -- 例如: {"can_upload_design": true, "can_approve_discount": false}
    
    -- 状态
    status VARCHAR(20) DEFAULT 'active',
    last_login DATETIME,
    last_login_ip VARCHAR(50),
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    CONSTRAINT uk_dealer_user UNIQUE (dealer_id, username)
);

CREATE INDEX idx_dealer_user_dealer ON dealer_user(dealer_id);
CREATE INDEX idx_dealer_user_role ON dealer_user(role);
```

### 4.2 客户管理（经销商视角）

```sql
-- 经销商客户表
CREATE TABLE dealer_customer (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_no VARCHAR(50) NOT NULL COMMENT '客户编号：DEALERCODE+YYYYMMDD+序号',
    dealer_id UUID NOT NULL REFERENCES dealer(id),
    
    -- 基本信息
    name VARCHAR(100) NOT NULL,
    phone VARCHAR(20) NOT NULL,
    alt_phone VARCHAR(20),
    province VARCHAR(50),
    city VARCHAR(50),
    district VARCHAR(50),
    address VARCHAR(500),
    
    -- 来源
    source VARCHAR(50) COMMENT '来源：门店/电话/网络/转介绍',
    
    -- 状态
    status VARCHAR(20) DEFAULT 'new' COMMENT '状态：new/following/ordered/completed',
    
    -- 关联工厂客户
    factory_customer_id UUID REFERENCES customer(id) COMMENT '关联工厂客户ID',
    
    -- 跟进
    last_follow_date DATETIME,
    next_follow_date DATE,
    designer_id UUID REFERENCES dealer_user(id),
    salesperson_id UUID REFERENCES dealer_user(id),
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    CONSTRAINT uk_dealer_customer_no UNIQUE (dealer_id, customer_no)
);

-- 经销商客户跟进
CREATE TABLE dealer_customer_follow (
    id UUID PRIMARY KEY,
    customer_id UUID NOT NULL REFERENCES dealer_customer(id),
    follow_type VARCHAR(20),
    follow_content TEXT,
    next_plan TEXT,
    next_follow_date DATE,
    attachment JSONB,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    created_by UUID REFERENCES dealer_user(id)
);
```

## 五、经销商财务模块

### 5.1 应收/应付结构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         经销商财务架构                                        │
└─────────────────────────────────────────────────────────────────────────────┘

【工厂视角】
│
├── 应收经销商款项
│   ├── 订单款（经销商下单时的应付）
│   ├── 样品款
│   └── 其他费用
│
└── 应付经销商款项
    ├── 佣金/返利
    └── 退补款

【经销商视角】
│
├── 应收客户款项
│   ├── 订单订金
│   ├── 进度款
│   └── 尾款
│
└── 应付工厂款项
    ├── 订单款（给工厂的）
    ├── 样品款
    └── 其他费用
```

### 6.2 接口签名示例

```javascript
// API签名算法
const crypto = require('crypto');

function signRequest(params, secretKey) {
  // 1. 排序参数
  const sortedParams = Object.keys(params)
    .sort()
    .reduce((acc, key) => {
      acc[key] = params[key];
      return acc;
    }, {});
  
  // 2. 拼接字符串
  const queryString = Object.entries(sortedParams)
    .map(([key, value]) => `${key}=${value}`)
    .join('&');
  
  // 3. HMAC签名
  const sign = crypto
    .createHmac('sha256', secretKey)
    .update(queryString)
    .digest('hex');
  
  return sign;
}

// 请求示例
const params = {
  api_key: 'DEALER_API_KEY_XXX',
  timestamp: Date.now(),
  method: 'POST',
  path: '/dealer/v1/orders',
  body: JSON.stringify({ customer_id: 'xxx', items: [...] })
};

const signature = signRequest(params, 'SECRET_KEY_XXX');
```
