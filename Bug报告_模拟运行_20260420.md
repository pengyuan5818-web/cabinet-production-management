## 🔴 严重 Bug（系统级故障，阻塞生产）

### Bug #1: `order_master` 缺少 3 个关键字段

**影响范围：** P2-2 智能排程完全不可用

| 缺失字段 | 类型 | 影响 |
|---------|------|------|
| `priority` | varchar | `generateSchedule()` SQL 报错 500 |
| `schedule_status` | varchar | `enqueue()` 更新失败，整个排程队列失效 |
| `estimated_hours` | numeric | 无法计算单个订单工时，排程结果不准确 |

**实测报错：**
```
[500] POST /api/production/schedule/generate
=> "字段 om.priority 不存在"
```

**根因：** 迁移 007（`production_scheduling_queue`）定义了 `order_master` 的扩展字段 `priority`、`schedule_status`、`estimated_hours`，但迁移未执行。

**修复：**
```sql
ALTER TABLE order_master ADD COLUMN priority VARCHAR(20) DEFAULT 'normal';
ALTER TABLE order_master ADD COLUMN schedule_status VARCHAR(20) DEFAULT 'unscheduled';
ALTER TABLE order_master ADD COLUMN estimated_hours NUMERIC(10,2);
```

**影响：** 前端请求 404，但后端对应功能存在（只是路径不同）

| 前端错误路径 | 实际正确路径 | 备注 |
|------------|------------|------|
| `GET /api/cost/allocation/pool` | `GET /api/cost/overhead-pool` | 路由注册在 `/cost/overhead-pool` |
| `GET /api/dealers/commissions` | `GET /api/dealers/commissions/summary` | `/commissions` 被 `/:id` 匹配，id="commissions" |
| `GET /api/dealers/openapi/keys` | `GET /api/dealers/:id/api-key` | dealer.js 无 openapi 前缀路由 |

**Bug#4-1 实测：**

**Bug#4-2 实测：**
```
[500] GET /api/dealers/commissions
=> "无效的类型 uuid 输入语法: \"commissions\""
→ /:id 路由匹配，id="commissions" 不是 UUID
[200] GET /api/dealers/commissions/summary => ✅
```

### Bug #5: `commissionService.autoGenerate` 缺少 Webhook 事件

**影响范围：** P1-3 佣金结算

**根因：** `commission.created` 事件类型未在 `EVENT_TYPES` 中定义，`autoGenerate` 调用 `trigger(dealer_id, 'commission.settled', ...)` 但此时佣金状态是 `pending`（未结算），语义错误。

**EVENT_TYPES 已有：**
```javascript
'order.status_changed', 'order.production_stage', 'order.shipped',
'payment.confirmed', 'payment.rejected',
'commission.settled',  // ← 只有"已结算"，无"已生成"
'invoice.created', 'invoice.issued'
```

**正确语义：**
- 佣金生成（pending）→ 应该是 `commission.created`
- 佣金结算（settled）→ `commission.settled` ✅

## 🔵 轻微 Bug（数据/行为问题）

### Bug #6: 认证中间件路径问题（已Workaround）

**现象：** 使用 `Authorization: <token>`（无 Bearer）时所有 API 返回 401。

**根因：** `authMiddleware` 要求 `Authorization: Bearer <token>` 格式。

**说明：** 这不是 bug，是正确行为。但早期诊断中未意识到导致浪费了大量调试时间。**前后端接口文档应明确标注必须带 `Bearer` 前缀。**

### Bug #8: `webhook` 表缺少 `events` 数组字段

**影响：** 无法支持一个 Webhook 订阅多个事件。

**实测：** `webhook.events` 字段不存在，`webhook` 表只有 `event_type`（单值）。

**当前设计：** 一个 Webhook 只支持一种事件类型。如需多事件需要额外部署多个 Webhook。
